2FA AUTHENTICATION FLOW IMPLEMENTED
═══════════════════════════════════════════════════════════════════════════════

STEP 1: LOGIN
├─ Screen: LoginActivity (enhanced)
├─ Input: username, email, password
├─ API: POST /auth/login
├─ Response: {loginOk, otpToken, email}
└─ Action: Save otpToken & email

STEP 2: CHOOSE METHOD
├─ Screen: AuthMethodSelectionActivity (NEW)
├─ Display: User's email address
├─ Option A: Email Verification
│  └─ API: POST /auth/send-otp {otpToken}
│  └─ Navigate: → OTPVerificationActivity
└─ Option B: Dev Mode (if enabled)
   └─ API: POST /auth/dev-login {otpToken}
   └─ Navigate: → HomeActivity directly

STEP 3: OTP VERIFICATION (Email Path)
├─ Screen: OTPVerificationActivity (NEW)
├─ Input: 6-digit code from email
├─ Features:
│  ├─ 5-minute countdown timer
│  ├─ 3 attempt limit
│  ├─ Resend button (30-sec cooldown)
│  ├─ Back button to return
│  └─ Error handling with messages
├─ API: POST /auth/verify-otp {otpToken, code}
├─ Response: {token, username}
└─ Action: Save JWT, clear otpToken, navigate to home

STEP 4: HOME PAGE (Authenticated)
├─ JWT token saved and valid
├─ OTP token cleared
├─ User can access app features
└─ Token persists across app restarts

API ENDPOINTS IMPLEMENTED
═══════════════════════════════════════════════════════════════════════════════

Authentication Endpoints (5):

1. POST /auth/register
   Request: {username, email, password}
   Response: {message}
   Purpose: Register new user account

2. POST /auth/login
   Request: {username, email, password}
   Response: {loginOk, otpToken, email}
   Purpose: Login user, get temporary OTP token (10 min)

3. POST /auth/send-otp
   Request: {otpToken}
   Response: {otpSent, email}
   Purpose: Send 6-digit OTP code to registered email

4. POST /auth/verify-otp
   Request: {otpToken, code}
   Response: {token, username}
   Purpose: Verify OTP code, receive JWT token (7 days)

5. POST /auth/dev-login
   Request: {otpToken}
   Response: {token, username, devMode}
   Purpose: Dev mode login (skip email verification)
   Note: Only available when ALLOW_DEV_LOGIN=true on server

TROUBLESHOOTING QUICK REFERENCE
═══════════════════════════════════════════════════════════════════════════════

Problem: "Cannot connect to server"
Solution: Update ApiConfig.BASE_URL to your server address

Problem: "Email not received"
Solution: Check SMTP configuration on backend, or use dev mode

Problem: "OTP code not working"
Solution: Use latest code sent, code expires after 5 minutes

Problem: "Invalid credentials"
Solution: Verify username/email/password are correct

Problem: "Too many attempts"
Solution: Wait or click "Resend Code" to get new OTP

Problem: "OTP token expired"
Solution: Login again, user has 10 minutes to complete verification

See: 2FA_TESTING_GUIDE.md for detailed troubleshooting
